# Daily Digest — Template

> **A daily intelligence system that turns what you read into a curated, targeted newsletter — written in your voice, filtered to what you actually care about.**

---

## What This Is

You read a lot. Newsletters pile up. Articles get saved and forgotten. Somewhere in that pile is the information you need to do your job better — but extracting it takes hours you don't have.

This tool automates that extraction. Every morning, it:

1. Pulls everything new from your Readwise Reader feed (newsletters, saved articles, RSS feeds)
2. Filters it against a profile you define — your portfolio companies, sectors, geographies, people
3. Sends it to Claude (Anthropic's AI) with your personal writing guide
4. Delivers a structured digest to your email and publishes it to a searchable web archive

The output isn't a summary. It's an analysis — written in the voice you configure, organized around what you care about, with inline citations back to every source.

---

## The Three Configuration Files

Everything that makes your digest yours lives in three places:

### 1. `config/writing_guide.md` — Your voice
Define a style reference (a writer whose analytical approach you admire), specify the output format (paragraphs vs. bullets, section structure), and set tone rules. This is the difference between a digest that sounds like a generic AI and one that sounds like something you'd actually write.

### 2. `config/filter_config.json` — Your world
List the entities you care about:
- **Portfolio companies** — the digest will surface every article with direct relevance and explain why it matters
- **Geographies** — flag news from or affecting specific regions
- **Industries and sectors** — track structural developments in your focus areas
- **Macro themes** — crosscutting patterns worth watching (e.g., deglobalization, AI infrastructure, energy transition)
- **People** — individuals whose moves are signal for you

### 3. `~/.daily_digest/config.json` — Your credentials
Created by `setup.py`. Stores your Readwise token, Anthropic API key, and email settings. Never committed to git.

## Quick Start

```bash
# 1. Clone this repository
git clone https://github.com/YOUR_GITHUB/daily-digest.git
cd daily-digest

# 2. Install dependencies
pip install -r requirements.txt

# 3. Run the setup wizard
python3 setup.py

# 4. Personalize your digest
#    Edit config/writing_guide.md — define your writing style
#    Edit config/filter_config.json — add your portfolio, sectors, geographies

# 5. Test it
python3 digest.py

# 6. Deploy your archive site to Netlify
#    See INSTALLATION.md → Netlify Deployment
```

## Architecture

```
digest.py           — Main script. Runs daily via cron or launchd.
build_site.py       — Builds digests.json for the web archive.
setup.py            — One-time setup wizard.
config/
  writing_guide.md  — Writing style instructions for Claude.
  filter_config.json — Your interests profile.
site/               — Static web archive (deployed to Netlify).
  index.html        — Digest listing with search and filter.
  digest.html       — Individual digest view.
  about.html        — About page.
  css/style.css     — Styles.
  js/app.js         — Client-side logic.
  data/digests.json — Generated by build_site.py. Gitignored by default.
netlify.toml        — Netlify build config.
```

### Data Flow

```
Readwise Reader (feed + saved articles)
        ↓
  digest.py fetches last 24h
        ↓
  filter_config.json marks priority docs
        ↓
  writing_guide.md + filter_config.json → prompt to Claude
        ↓
  Claude generates HTML digest
        ↓
  ┌─────────────┬──────────────────────┐
  ↓             ↓                      ↓
Email       Archive JSON          site/data/digests.json
delivered   saved locally         pushed to GitHub
                                        ↓
                                  Netlify auto-deploys
                                  web archive
```

## Scheduling (Daily Automation)

### GitHub Actions (recommended)
The repository includes `.github/workflows/daily-digest.yml`, which runs `digest.py` on GitHub's servers every morning — no local machine required. Setup requires adding five repository secrets (API keys and email credentials) in your fork's Settings → Secrets.

To trigger a digest manually at any time:
```bash
gh workflow run daily-digest.yml
```

See `INSTALLATION.md → Step 6` for the full setup walkthrough, including secrets configuration and timezone adjustment.

### Local schedule (macOS launchd / cron)
See `INSTALLATION.md → Step 6` for launchd and cron instructions if you prefer to run the digest locally.

## Web Archive & Subscribe

Your archive site deploys to Netlify automatically when you push to GitHub. It includes:
- A searchable, filterable listing of all past digests
- Full-text search across every digest
- Daily/Weekly filter tabs
- A subscribe form (handled by Netlify Forms — no backend required)

When someone subscribes via the form, their email is captured in your Netlify dashboard. You can then manually add them to your email sends or connect a tool like Zapier to automate it.

## What This Is Not

This tool does **not** replace reading. It surfaces what's worth reading and tells you why it matters to your specific situation. The links back to every source article are intentional — the digest is a triage layer, not a replacement for engaging with primary sources.
